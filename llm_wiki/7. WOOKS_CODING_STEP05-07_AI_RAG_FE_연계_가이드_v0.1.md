# WOOK'S CODING STEP05-07 AI/RAG 및 FE 연계 가이드 v0.1

## 1. 현재 구현 상태 분석

- STEP04 코드 실행 구조는 Django `/submissions/run/`, `/submissions/submit/`, `Submission`, `ExecutionJob`, 별도 Worker polling 구조를 유지한다.
- Django는 화면, 인증, 권한, CRUD, DB 로그 저장, FastAPI 프록시 역할을 담당한다.
- FastAPI는 AI/RAG 전용이다. Django는 OpenAI 또는 ChromaDB를 직접 호출하지 않는다.
- Django 공통 프록시 경로는 `/ai-proxy/*` 이다. FastAPI 내부 경로 `/ai/*` 와 충돌하지 않게 분리했다.
- AI 호출 로그는 `LLMRequestLog`, 실패 로그는 `ErrorLog`에 저장한다.
- ChromaDB collection은 MVP에서 `wrong_note_embeddings`만 사용한다.
- `concept_embeddings`, `pattern_embeddings`, GraphRAG, Redis/RabbitMQ/Celery는 MVP 범위에서 제외한다.

## 2. Django Endpoint 목록

| 기능 | Django Fetch URL | Method | FastAPI 내부 URL | request_type |
|---|---:|---:|---:|---|
| AI 힌트 요청 | `/ai-proxy/hint/` | POST | `/ai/hint` | `hint` |
| 유사 오답노트 검색 | `/ai-proxy/wrong-note/search/` | POST | `/ai/wrong-note/search` | `wrong_note_search` |
| 오답노트 AI 분석 | `/ai-proxy/wrong-note/analyze/` | POST | `/ai/wrong-note/analyze` | `wrong_note_analyze` |
| 오답노트 임베딩 | `/ai-proxy/wrong-note/embed/` | POST | `/ai/wrong-note/embed` | `wrong_note_embed` |
| 내 노트 질의응답 | `/ai-proxy/wrong-note/ask/` | POST | `/ai/wrong-note/ask` | `note_ask` |

모든 endpoint는 로그인 필요, CSRF 필요, JSON body 필요.
Django는 payload의 `user_id`를 현재 로그인 사용자 id로 덮어쓴다.

## 3. 공통 요청/응답 JSON 계약

### 공통 응답

```json
{
  "status": "success|failed|timeout|empty",
  "request_id": "django-generated-x-request-id",
  "message": "status or error message",
  "data": {}
}
```

### 실패 상태 기준

| status | 의미 | FE 처리 |
|---|---|---|
| `success` | 정상 결과 있음 | 결과 영역 렌더링 |
| `empty` | 요청은 성공했지만 검색 결과 없음 | empty 안내, 재시도 버튼은 선택 |
| `failed` | FastAPI 오류, ChromaDB 오류, 응답 포맷 오류 등 | 오류 메시지 표시, request_id 노출 |
| `timeout` | FastAPI 응답 제한시간 초과 | timeout 안내, 재시도 버튼 표시 |

## 6. AI/RAG 담당자 진행 가이드

FastAPI에서 구현 또는 보완할 endpoint:

- `POST /ai/hint`
- `POST /ai/wrong-note/search`
- `POST /ai/wrong-note/analyze`
- `POST /ai/wrong-note/embed`
- `POST /ai/wrong-note/ask`

필수 규칙:

- `X-Internal-API-Key` 검증 필수.
- `X-Request-ID` 없으면 400 반환.
- 모든 응답에 `request_id` 포함.
- ChromaDB collection은 `wrong_note_embeddings`만 사용.
- ChromaDB metadata에 `user_id`, `wrong_note_id`, `source` 포함.
- 검색 시 `where={"user_id": user_id}` 필터 필수.
- 검색 결과는 `note_id`, `source`, `score` 필수.
- 내 노트 질의응답은 `evidence_note_ids`, `scores` 필수.
- `RAG_SCORE_THRESHOLD=0.35`, `RAG_TOP_K=5` 기준.
- MVP에서 `concept_embeddings`, `pattern_embeddings` 생성 금지.
- 실제 AI 미구현 시 성공 stub 대신 `status=failed`, `message=not_implemented` 또는 명확한 구현 상태 반환.

## 7. 프론트 담당자 진행 가이드

Fetch 공통 헤더:

```js
{
  "Content-Type": "application/json",
  "X-CSRFToken": getCookie("csrftoken"),
  "Accept": "application/json"
}
```

공통 처리:

- 요청 시작 시 버튼 disabled, 상태 문구 loading.
- `status=success`: `data` 렌더링.
- `status=empty`: 결과 없음 문구 표시.
- `status=failed`: `message`와 `request_id` 표시.
- `status=timeout`: timeout 문구와 재시도 버튼 표시.
- HTTP 502여도 body의 공통 JSON 계약을 우선 사용.

연동 URL:

- 힌트: `POST /ai-proxy/hint/`
- 유사 오답 검색: `POST /ai-proxy/wrong-note/search/`
- 오답 분석: `POST /ai-proxy/wrong-note/analyze/`
- 임베딩: `POST /ai-proxy/wrong-note/embed/`
- 내 노트 질문: `POST /ai-proxy/wrong-note/ask/`

## 8. 검증 체크리스트

### BE 담당자

| 항목 | 확인 명령/화면 | 기대 결과 |
|---|---|---|
| URL 등록 | `docker compose exec -T django python manage.py check` | 오류 없음 |
| 마이그레이션 영향 없음 | `docker compose exec -T django python manage.py migrate --check` | 미적용 migration 없음 |
| AI proxy 테스트 | `docker compose exec -T django python manage.py test ai_proxy` | OK |
| LLM 로그 저장 | admin 또는 shell에서 `LLMRequestLog` 조회 | request_id/status/input/response 저장 |
| ErrorLog 저장 | FastAPI 중지 후 endpoint 호출 | `source=fastapi`, status failed/timeout |
| STEP04 회귀 | 문제 제출 후 `/submissions/<id>/result/` polling | 기존 run/submit/Worker 흐름 유지 |

### AI/RAG 담당자

| 항목 | 확인 명령/화면 | 기대 결과 |
|---|---|---|
| 인증 헤더 검증 | FastAPI 직접 호출에서 key 제거 | 401 |
| Request ID 검증 | FastAPI 직접 호출에서 X-Request-ID 제거 | 400 |
| 유저 필터 | 서로 다른 user_id 노트 검색 | 타 사용자 note 미노출 |
| Chroma collection | ChromaDB 확인 | `wrong_note_embeddings`만 사용 |
| 검색 결과 | `/ai/wrong-note/search` | note_id/source/score 반환 |
| 내 노트 질문 | `/ai/wrong-note/ask` | answer/evidence_note_ids/scores 반환 |

### FE 담당자

| 항목 | 확인 명령/화면 | 기대 결과 |
|---|---|---|
| 힌트 버튼 | 문제 풀이 화면 | loading 후 hint content 표시 |
| 오답 검색 카드 | 오답노트 작성 화면 | note_id/source/score 표시 |
| AI 분석 영역 | 오답노트 작성 화면 | problem_core/cause/solution/caution 표시 |
| 내 노트 질문 | `/wrongnotes/ask/` | answer/evidence 표시 |
| empty 처리 | 검색 결과 없는 질문 | 결과 없음 문구 |
| timeout 처리 | FastAPI 지연 또는 중지 | timeout/failed 문구와 request_id |
| PointToast | 제출/오답노트/복습 액션 | 지급 포인트 안내 |
