## 1. 전체 데이터 흐름 (Client – Server – DB – 외부 LLM)

```
┌──────────┐   HTTPS/HTTP    ┌───────────────────────── 앱 서버 (EC2 #1, Docker 네트워크) ─────────────────────────┐
│  사용자   │───────────────▶│                                                                                     │
│ (브라우저)│                │  ┌────────┐   ┌──────────────────────────┐   내부호출     ┌──────────────────────┐  │
│  Fetch   │◀───────────────│  │ Nginx  │──▶│  Django (Gunicorn/WSGI)  │──────────────▶│  FastAPI (/ai/*)     │  │
└──────────┘   JSON/HTML    │  │ :80    │   │  웹·인증·권한·CRUD·프록시 │  X-Internal-  │  AI/RAG (stateless)  │  │
                            │  └───┬────┘   │  (/ai-proxy/* )           │  API-Key       └────┬────────────┬────┘  │
                            │      │static  └────────┬─────────────────┘  X-Request-ID       │            │       │
                            │      │                 │ 쓰기/조회                    (RAG 쓰기) │   OpenAI   │       │
                            │  [staticfiles vol]     │                                        ▼   (외부 LLM)│       │
                            │                        │                              ┌──────────────┐  ┌─────▼─────┐ │
                            │                        │                              │  ChromaDB    │  │  OpenAI   │ │
                            │                        │                              │ (벡터, 컨테이너)│  │ gpt-4o-mini│ │
                            │                        ▼                              └──────────────┘  │ embedding │ │
                            └────────────────────────┼──────────────────────────────────────────────  └───────────┘ │
                                                     │ (RDS 연결)                                                    │
                            ┌────────────────────────▼─────────────┐   poll (FOR UPDATE SKIP LOCKED)   ┌────────────┐│
                            │      AWS RDS (PostgreSQL, 원천)        │◀─────────────────────────────────│  Worker    ││
                            │  users·problems·submissions·jobs·     │      결과 write                   │ (EC2 #2)   ││
                            │  wrong_notes·coding_state·logs …      │─────────────────────────────────▶│ 코드 격리실행│
                            └───────────────────────────────────────┘                                  └────────────┘│
                                                                                                                     │
                            └─────────────────────────────────────────────────────────────────────────────────────┘
```

**흐름 설명**
1. 사용자 요청은 **Nginx(:80)** 만 외부에 노출. 정적파일은 Nginx가 볼륨에서 직접 서빙, 그 외는 Django로 프록시.
2. **Django**가 인증·권한·CRUD를 처리하고 PostgreSQL(RDS)의 **쓰기 주인**.
3. AI 요청은 Django `/ai-proxy/*` → 내부망 **FastAPI `/ai/*`** (헤더 인증). FastAPI는 **stateless**(RDB 직접 접근 없음), payload로만 데이터 수신.
4. FastAPI가 **ChromaDB**(벡터 쓰기 주인)와 **OpenAI**(외부 LLM)를 호출.
5. 코드 실행은 **Worker(별도 EC2)** 가 `jobs` 테이블을 폴링(`FOR UPDATE SKIP LOCKED`)해 격리 실행 후 결과를 RDS에 기록.

**데이터 소유권(SSOT)**

| 컴포넌트 | PostgreSQL(RDS) | ChromaDB | 외부 LLM |
|---|:--:|:--:|:--:|
| Django | ✅ 쓰기 주인 | ❌ | ❌(프록시만) |
| FastAPI | ❌ (stateless) | ✅ 쓰기 주인 | ✅ 호출 |
| Worker | ✅ jobs 실행자 | ❌ | ❌ |

## 2. 배포 아키텍처 (Nginx – Gunicorn – Django/WSGI)

```
   [외부]                     [앱 서버 EC2 #1 — compose/main.yml]
                    ┌───────────────────────────────────────────────────────────┐
 브라우저 ─:80──▶  │  Nginx (nginx:1.27-alpine)                                  │
                    │   location /static/ → alias /staticfiles/ (volume, expires 7d)
                    │   location /       → proxy_pass http://django:8000         │
                    │                      X-Forwarded-Proto/For/Host 전달        │
                    │        │                                                    │
                    │        ▼ (내부)                                             │
                    │  Django 컨테이너                                            │
                    │   command: migrate --no-input                              │
                    │          → collectstatic --noinput                         │
                    │          → gunicorn config.wsgi:application --bind :8000    │  ← WSGI
                    │   healthcheck: GET /health/ (5s×20)                        │
                    │        │                                                    │
                    │        ▼ (내부, http://fastapi:8001)                        │
                    │  FastAPI 컨테이너 (uvicorn) ── ChromaDB 컨테이너            │
                    └───────────────────────────────────────────────────────────┘
                              │ (RDS 5432, 내부/보안그룹 제한)
                              ▼
                         AWS RDS PostgreSQL
```

- **WSGI 스택**: 요청 = Nginx(리버스 프록시) → Gunicorn(WSGI 서버) → Django(`config.wsgi:application`).
- **정적 파일**: `collectstatic` 결과를 `staticfiles` named volume에 적재 → Nginx가 `/static/`으로 서빙(앱 부하 분리).
- **기동 순서 보장**: Nginx는 `depends_on: django(service_healthy)` 조건으로만 뜸(무중단성↑).
- **워커 분리**(`compose/worker.yml`): Worker는 앱 서버와 다른 EC2에서 단독 기동(`restart: unless-stopped`).
- **로컬 개발용**(`docker-compose.yml`): postgres·chromadb·django·fastapi·worker·nginx 전체를 한 번에(운영은 RDS라 postgres 컨테이너 제외).

## 3. 클라우드 / 컨테이너 구성 (AWS + Docker 네트워크)

```
                             ┌─────────────────────── AWS ───────────────────────┐
                             │                                                    │
   Internet ──▶ (SG: 80/443 허용, 22 제한)                                        │
        │                    │   ┌──────────────── EC2 #1 : 앱 서버 ───────────┐  │
        └───────────────────▶│   │  Docker network (bridge)                    │  │
                             │   │   ┌───────┐ ┌────────┐ ┌────────┐ ┌───────┐ │  │
                             │   │   │ nginx │ │ django │ │fastapi │ │chroma │ │  │
                             │   │   │  :80  │ │ :8000  │ │ :8001  │ │ :8000 │ │  │
                             │   │   └───────┘ └────────┘ └────────┘ └───────┘ │  │
                             │   │   volumes: staticfiles, chromadata,          │  │
                             │   │            ./volumes/logs(bind), data(ro)    │  │
                             │   └──────────────────┬───────────────────────────┘  │
                             │                      │ (SG: RDS 5432, 앱/워커 SG만)   │
                             │   ┌──────────────────▼───────────┐                   │
                             │   │  AWS RDS (PostgreSQL)         │                   │
                             │   │  ap-northeast-2, 원천 데이터   │                   │
                             │   └──────────────────▲───────────┘                   │
                             │                      │                                │
                             │   ┌──────────────────┴─ EC2 #2 : 워커 서버 ────────┐  │
                             │   │  Docker: worker (코드 격리 실행, jobs polling)  │  │
                             │   └─────────────────────────────────────────────────┘ │
                             └────────────────────────────────────────────────────┘
```

- **AWS 리소스**: EC2 ×2(앱/워커) + RDS(PostgreSQL). 정적 파일은 Nginx가 Docker 볼륨에서 서빙(S3 미사용, §4.2 확장 포인트).
- **Docker 서비스**(앱 서버): `nginx`·`django`·`fastapi`·`chromadb` — 동일 bridge 네트워크에서 **서비스명 DNS**(`django:8000`, `fastapi:8001`, `chromadb`)로 통신.
- **볼륨/영속성**:
  - named volume: `staticfiles`(정적), `chromadata`(벡터) → `down`엔 유지, **`down -v`에서만 삭제**.
  - bind mount: `./volumes/logs → /logs`(로그 호스트 보존), `./volumes/data → /app/data:ro`(seed 읽기전용).
- **환경변수**: 루트 `.env` 단일 통합, compose `env_file: .env`로 전 컨테이너 주입.

## 4. 보안 / 확장성 고려

### 4.1 보안 (계층 방어)

| 계층 | 조치 |
|---|---|
| 네트워크(SG) | 외부 노출은 **Nginx :80 하나**. FastAPI/Worker/ChromaDB/RDS는 내부망 전용. RDS 5432는 앱·워커 보안그룹에서만 접근을 허용한다. |
| 내부 API 인증 | Django→FastAPI 호출은 `X-Internal-API-Key`(불일치 401) + `X-Request-ID`(추적) 필수. FastAPI `Depends(verify_internal)`로 검증 |
| 인증/세션 | 커스텀 유저 세션 인증, `@login_required`/`LoginRequiredMixin`, CSRF 미들웨어 + Fetch `X-CSRFToken` |
| 권한 분리 | 역할(`role`)·`is_service_admin`로 운영 화면 분리, 오답노트/RAG는 `user_id`로 **소유자 격리** |
| 코드 실행 격리 | 신뢰 불가 사용자 코드는 **별도 EC2의 Worker 컨테이너**에서만 실행(타임아웃 5s) → 웹/DB 서버 보호 |
| 데이터 격리 | FastAPI stateless(RDB 미접근), 벡터 컬렉션 `user_id` where 필터 |
| 비밀/로그 | `.env` git 제외, LLM 감사(`LLMRequestLog`)·오류(`ErrorLog`) 기록, PII truncate |

### 4.2 확장성

- **역할 분리(수평 확장 용이)**: Django(서비스) ↔ FastAPI(AI) ↔ Worker(실행)를 분리 → 각자 독립 스케일아웃/재시작. FastAPI stateless라 인스턴스 증설 자유.
- **워커 스케일**: `jobs` 테이블 `FOR UPDATE SKIP LOCKED` 선점 → 워커 N대로 수평 확장 가능(중복 실행 없음).
- **벡터 임베더 버전관리**: 컬렉션 `wrong_note_embeddings-<sig>`로 임베더 교체 시 차원 충돌 없이 격리.
- **확장 포인트**: ① 정적/미디어 → **S3 + CloudFront** 이전(현 Nginx 볼륨 → CDN), ② Worker 큐를 Redis/RabbitMQ로 승격(현 폴링), ③ HTTPS/ALB 종단 + 오토스케일링 그룹.
