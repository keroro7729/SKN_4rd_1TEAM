## 1. 화면 설계 개요

- **아키텍처**: Django 5 **MVT**(Model–View–Template) 서버 렌더링 + **부분 비동기(Fetch)**. 페이지 골격은 서버 템플릿, LLM/코드실행 등 대기가 긴 상호작용만 클라이언트 Fetch로 처리.
- **공통 레이아웃**: `templates/base.html`이 헤더(로고·네비·프로필 pill)·메인·푸터·**전역 미니튜터 챗봇**·프로필 모달을 감싼다. 모든 페이지가 `{% extends "base.html" %}`.
- **디자인 컨셉**: "코딩테스트 RPG" — 포인트/레벨/동물 아바타 게이미피케이션이 헤더·마이페이지에 상시 노출.
- **정적 자원**: `static/css/{core,pages}/*.css`(레이어 분리) + `static/js/*.js`(페이지별 스크립트). 버전 쿼리스트링(`?v=...`)으로 캐시 무효화.

## 3. 반응형 UI 설계 (Flex/Grid)

**전략**: 모바일 우선은 아니지만, **Grid 우선 + Flex 보조 + `@media` 브레이크포인트**로 데스크톱/태블릿/모바일을 커버.

- **뷰포트**: `base.html`에 `<meta name="viewport" content="width=device-width, initial-scale=1.0">`.
- **레이아웃 실측(CSS 통계)**: `display:grid` 약 145곳, `display:flex` 약 107곳, `@media` 쿼리 53곳, `grid-template*` 178곳, `minmax()` 133곳, `flex-wrap` 36곳 → **Grid/Flex 기반이 설계의 근간**임을 정량 확인.
- **핵심 반응형 패턴**:
  - 문제 풀이(S-06): `code-column`(에디터) + `sidekick-column`(AI 힌트/튜터) **2단 Grid** → 좁은 화면에서 1단으로 접힘.
  - 문제 목록(S-05)·마이페이지(S-14): 카드 `grid-template-columns: repeat(auto-fill, minmax(...))`로 화면폭에 따라 열 수 자동 조정.
  - 헤더 네비/프로필 pill: Flex + `flex-wrap`으로 축소 시 줄바꿈.
- **오버플로 방어**: `pages/layout_overflow_fix.css`로 가로 스크롤/클리핑 방지(커밋 `9248ebf`, `727cb72`에서 반응형 안정화).

> 브레이크포인트별 목업/치수 상세는 후속 보강 항목(현재는 실 CSS 규칙으로 근거 제시).

## 4. 인증/권한 흐름 설계 (화면 단위)

**커스텀 유저**: `AUTH_USER_MODEL = accounts.CustomUser` (필드 `role`·`point`·`is_subscribed`). `LOGIN_URL = "accounts:login"`.

### 4.1 접근 제어 계층

```
비로그인 ──▶ 공개(홈·로그인·회원가입)만 접근
   │  보호 URL 접근 시
   ▼
@login_required / LoginRequiredMixin ──▶ LOGIN_URL(accounts:login)로 리다이렉트(?next=원래경로)
   │  로그인 성공(세션 생성)
   ▼
일반 사용자 ──▶ 문제·오답노트·마이페이지 (본인 데이터만: queryset을 user로 필터)
   │  user.is_service_admin == True 인 경우만
   ▼
관리자 ──▶ /adminpanel/* (대시보드·로그·AI 실험실)
```

- **화면 단위 가드**:
  - CBV 페이지: `LoginRequiredMixin`(+ 오답노트는 `queryset.filter(user=self.request.user)`로 **소유자 격리**).
  - FBV 액션(코드실행·AI 프록시): `@login_required @require_POST` 데코레이터 이중 가드.
  - 관리자 화면: `base.html` 네비에서 `{% if user.is_service_admin %}`로 진입점 숨김 + View에서 권한 재검증.
- **템플릿 분기**: `base.html`이 `user.is_authenticated`로 네비(로그인/회원가입 ↔ 문제풀이/오답노트/마이페이지·로그아웃)와 프로필 pill·챗봇 노출을 전환.
- **CSRF**: 모든 POST 폼에 `{% csrf_token %}`, Fetch 요청은 `X-CSRFToken` 헤더로 쿠키값 전송(§5).
- **계정 보호 흐름**: 계정 상세/비밀번호/탈퇴는 `AccountVerifyView`로 **재인증 후 진입**(S-16).

## 5. LLM 상호작용 UX 설계 (비동기 · 로딩 · 예외)

LLM/코드실행은 응답이 수 초 걸리므로 **입력 → 로딩 → 응답/에러**의 3-상태를 명시적으로 설계하고, 전부 **Fetch 비동기**로 처리(페이지 리로드 없음).

### 5.1 AI 단계별 힌트 (S-06 사이드바)

| 상태 | UI |
|---|---|
| 초기 | 힌트 아코디언 접힘, 토글 `disabled`, "다음 힌트 요청" 버튼 |
| 요청 중 | 버튼 비활성, 로딩 표시 |
| 응답 | 레벨 1→2→3 순차 해금, 해당 `hint-step-body`에 힌트 렌더 |
| 에러/타임아웃 | 사용자 친화 메시지 표시(정답 노출 없음) |

- 데이터 소스: `problem_solve.html`의 `data-hint-url="{% url 'ai_proxy:hint' %}"` → `problem_solve.js`가 Fetch. 서버가 문제 지문·`coding_state`·`level`을 enrich(§백엔드).

### 5.2 코드 실행/제출 (비동기 Job + 폴링)

가장 정교한 비동기 UX. **작업 생성 → 상태 폴링 → 결과 렌더**의 Job 패턴(`problem_solve.js`).

```
[실행/제출 클릭]
   └▶ run/submit 버튼 disabled, #run-loading-box 표시("대기열 등록")
   └▶ fetch(run|submit URL, POST, X-CSRFToken)  → { submission_id }
        └▶ pollResult(id): 1초 간격 fetch(/submissions/<id>/result/)
             ├─ pending/running → 로딩 유지, 재폴링
             ├─ success/wrong/failed/timeout → 결과 탭(stdout/기대출력/케이스) 렌더
             └─ (try/catch/finally) 오류 시 메시지, finally에서 버튼 재활성화
```

### 5.3 미니튜터 챗봇 (G-01 전역 플로팅)

`base.html` + `chatbot.js`. 좌하단 FAB → 패널 토글.

| 상태 | UI |
|---|---|
| 대기 | placeholder "궁금한 점을 입력하면…" |
| 전송 중 | 전송버튼 `disabled`, **타이핑 인디케이터**(`● ● ● 생각하는 중`) 말풍선 |
| 응답 | 최신 답변 1개 말풍선(Q 에코 + A 버블), `escapeHtml`로 XSS 방어 |
| 에러 | catch에서 "답변을 불러오지 못했어요" 표시, finally에서 입력 재활성화 |

- 대화 이력(`history`, 최근 8턴)·현재 활동(`path`·`title`·`problem_id`)을 **클라이언트가 수집해 매 요청 동봉** → 서버 stateless.

### 5.4 오답노트 AI 분석 (S-11 작성 / S-12 상세)

- 작성(S-11): 회고 코멘트 입력 → **AI 6섹션 분석**(문제핵심·정석풀이·오답원인·개선·피드백·체크리스트) Fetch 호출 → 로딩 후 섹션 렌더 → 저장 시 ChromaDB 임베딩.
- 상세(S-12): 저장된 AI 분석 + **유사 오답노트(RAG)** 회수 결과 표시(본인 노트만).

### 5.5 공통 LLM UX 원칙

1. **낙관적 비활성화**: 요청 시작 시 트리거 버튼 `disabled` → 중복요청 방지, `finally`에서 항상 복구.
2. **명시적 로딩 신호**: 로딩 박스/타이핑 인디케이터로 "처리 중"을 시각화.
3. **graceful 에러**: 실패/타임아웃 시 500을 노출하지 않고 사용자 메시지로 강등(서버 계약이 `status`로 통일).
4. **응답 이스케이프**: LLM 출력은 `escapeHtml` 후 삽입.
