## 2. 검증 환경 준비

- Git, Docker Engine, Docker Compose v2 이상, 웹 브라우저, OpenAI API 키
- 로컬 여유 포트: 80, 5432

```bash
git clone https://github.com/SKNETWORKS-FAMILY-AICAMP/SKN29-4TH-1Team.git
cd SKN29-4TH-1Team
git switch main
git rev-parse HEAD
cp .env.example .env
```

기준 SHA는 `3fc8d99704a6931d6331e2758e6efd95d0b6fb54`다. 이후 커밋은 실제 SHA를 결과표에 기록한다.

`.env` 필수 확인값:

```dotenv
DJANGO_SECRET_KEY=<긴 랜덤 문자열>
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1
POSTGRES_DB=wooks_coding
POSTGRES_USER=wooks
POSTGRES_PASSWORD=<검증용 비밀번호>
INTERNAL_API_KEY=<Django와 FastAPI 공용 키>
OPENAI_API_KEY=<실제 API 키>
CODE_TIMEOUT_SEC=5
CODE_MEM_LIMIT_MB=256
CODE_DISABLE_NETWORK=true
```

합격 기준: `.env`는 Git 추적 대상이 아니며 예시 비밀값을 사용하지 않는다. AI 품질 검증 시 실제 `OPENAI_API_KEY`를 설정한다.

## 3. 정적·구성 검증

```bash
python -m compileall -q django fastapi_app worker
docker compose --env-file .env -f docker-compose.yml config --quiet
docker compose --env-file .env -f compose/main.yml --project-directory . config --quiet
docker compose --env-file .env -f compose/worker.yml --project-directory . config --quiet
```

합격 기준: 네 명령 모두 종료 코드 0.

```bash
docker compose --env-file .env -f compose/main.yml --project-directory . config --services
docker compose --env-file .env -f compose/worker.yml --project-directory . config --services
```

합격 기준:

- 앱 서버: `chromadb`, `django`, `fastapi`, `nginx`만 포함
- Worker 서버: `worker`만 포함
- 운영 시 두 서버가 동일한 RDS `POSTGRES_HOST` 사용

## 4. 로컬 통합 기동

현재 `compose/local.yml`은 없으므로 루트 `docker-compose.yml`을 사용한다.

```bash
docker compose --env-file .env -f docker-compose.yml up --build -d
docker compose --env-file .env -f docker-compose.yml ps
```

합격 기준:

- `postgres`, `chromadb`, `django`, `fastapi`, `worker`, `nginx`가 실행 상태다.
- Django의 `migrate`, `collectstatic`이 성공한다.
- `http://localhost/`에 접속된다.

장애 확인:

```bash
docker compose --env-file .env -f docker-compose.yml logs --tail=200 django
docker compose --env-file .env -f docker-compose.yml logs --tail=200 fastapi
docker compose --env-file .env -f docker-compose.yml logs --tail=200 worker
docker compose --env-file .env -f docker-compose.yml logs --tail=100 nginx
```

## 5. DB·초기 데이터·자동 테스트

```bash
docker compose --env-file .env -f docker-compose.yml exec -T django python manage.py check
docker compose --env-file .env -f docker-compose.yml exec -T django python manage.py migrate --check
docker compose --env-file .env -f docker-compose.yml exec django python manage.py createsuperuser
docker compose --env-file .env -f docker-compose.yml run --rm django python manage.py load_problems
docker compose --env-file .env -f docker-compose.yml exec -T django python manage.py test
```

합격 기준:

- Django system check 오류와 미적용 마이그레이션이 없다.
- 관리자 계정과 문제·카테고리·태그가 생성된다.
- 전체 테스트가 `OK`로 종료된다.

필수 자동 테스트 영역:

- AI 프록시 응답 계약과 요청 로그
- 코딩 상태 갱신·스킵·배치 처리
- 포인트 중복 방지와 미션 진행
- 취약 태그 기반 문제 추천
- 실행·제출 작업 생성, 결과 폴링, 테스트케이스 생성 대기
- 오답노트 생성 권한, 소유자 격리, 복습 포인트 중복 방지

## 6. 헬스체크·내부망

```bash
curl -fsS http://localhost/health/
docker compose --env-file .env -f docker-compose.yml exec -T fastapi python -c "import urllib.request; print(urllib.request.urlopen('http://127.0.0.1:8001/ai/health').read().decode())"
```

합격 기준: Django는 `{"status":"ok"}`, FastAPI는 정상 상태 JSON을 반환한다.

추가 합격 기준:

- 호스트에서 FastAPI 8001, ChromaDB 8000, Worker에 직접 접근할 수 없다.
- 로컬 PostgreSQL 5432 노출은 개발 환경에서만 허용한다.
- 보호된 FastAPI API는 내부 키와 요청 추적 ID가 없으면 401 또는 403을 반환한다.

## 7. 기능 검증 시나리오

| ID | 절차 | 합격 기준 |
|---|---|---|
| VT-AUTH-01 | 회원가입 후 로그인·로그아웃 | 계정 생성, 로그인 성공, 로그아웃 후 보호 화면 차단 |
| VT-AUTH-02 | 일반 사용자로 `/adminpanel/` 접근 | 관리자 기능 접근 거부 |
| VT-ACC-01 | 계정 상세 직접 접근 후 이메일 변경 | 비밀번호 재확인 전 차단, 확인 후 변경 및 이력 저장 |
| VT-PROB-01 | 난이도·상태·검색 필터 조합 | 조건에 맞는 활성 문제만 표시, 페이지 이동 시 조건 유지 |
| VT-PROB-02 | 즐겨찾기 연속 토글 | 추가·해제 정상 반영, 중복 데이터 없음 |
| VT-EXEC-01 | 정답 코드를 `실행` | HTTP 202 후 폴링 종료, 빠른 테스트 1건 결과 표시 |
| VT-EXEC-02 | 정답 코드를 `제출` | 전체 테스트 통과, 결과 `success`, 정답 포인트 1회 지급 |
| VT-EXEC-03 | 오답 코드 제출 | 결과 `wrong`, 통과 수와 오답노트 작성 링크 표시 |
| VT-EXEC-04 | 예외 발생 코드 제출 | 결과 `error`, 제한된 사용자용 오류 표시 |
| VT-EXEC-05 | 무한 반복 코드 제출 | 제한시간 내 `timeout` 종료 |
| VT-EXEC-06 | 테스트케이스 없는 문제 최초 실행 | `generating` 후 생성·실행 전환, 중복 생성 없음 |
| VT-WN-01 | 오답 최종 제출로 오답노트 작성 | 회고·AI 6섹션 분석·유사 노트 저장 및 인덱싱 |
| VT-WN-02 | 실행 결과 또는 정답 제출로 노트 작성 | HTTP 400 또는 화면 차단 |
| VT-WN-03 | 다른 사용자의 오답노트 URL 접근 | 404 또는 접근 거부 |
| VT-WN-04 | 복습 완료를 두 번 요청 | 최초 요청만 10P 지급 |
| VT-WN-05 | 노트 숨김 후 복원 | 삭제 없이 숨김 탭 이동 후 복원 |
| VT-RAG-01 | 사용자 A의 노트 내용으로 사용자 B가 검색 | A의 노트가 B 결과에 포함되지 않음 |
| VT-RAG-02 | 미니튜터에 내 오답 관련 질문 | 본인 노트 검색 근거를 활용해 답변, 교차 사용자 노트 미포함, 질의 로그 저장 |
| VT-AI-01 | 문제 풀이 중 단계별 힌트 요청 | 문제·코드·수준 반영, 정답 코드 직접 노출 없음 |
| VT-AI-02 | 미니튜터 연속 질의 | 대화 이력·현재 활동·코딩 상태·최근 오답 반영 |
| VT-GAME-01 | 제출·정답·오답노트·복습 반복 | 규정 포인트 지급, 중복 지급 없음, 레벨 갱신 |
| VT-ADMIN-01 | 관리자 대시보드·로그·AI 실험실 조회 | 집계·로그·그래프 조회 및 단일 노드 실행 성공 |

## 8. 코드 실행 보안 검증

| 공격·오류 유형 | 예시 | 합격 기준 |
|---|---|---|
| 무한 반복 | `while True: pass` | 시간초과 종료 |
| 과도한 메모리 | 무한 리스트 확장 | 메모리 제한 또는 오류 종료, Worker 생존 |
| 파일 쓰기 | 파일 생성·대용량 쓰기 | 파일 크기 제한으로 차단 |
| 외부 네트워크 | `socket` 연결 또는 URL 요청 | 네트워크 차단 오류 |
| 과도한 출력 | 무한 또는 대량 출력 | 저장 길이 제한, DB·화면 정상 |
| 프로세스 오류 | 예외 발생 | 내부 정보 대신 제한된 오류 반환 |

추가 합격 기준:

- 사용자 코드는 웹 서버가 아닌 Worker에서만 실행된다.
- 공격 코드 실행 후 Worker가 다음 작업을 처리한다.
- 비밀 환경변수, 내부 IP, DB 접속정보가 사용자 출력에 노출되지 않는다.

## 11. 운영 배포 검증

필수 설정:

- `DJANGO_DEBUG=False`
- 운영용 `DJANGO_SECRET_KEY`, `INTERNAL_API_KEY`, `OPENAI_API_KEY`
- 실제 도메인 또는 EIP를 `DJANGO_ALLOWED_HOSTS`에 지정
- 앱·Worker 서버의 `POSTGRES_HOST`를 동일 RDS 엔드포인트로 지정
- RDS 5432는 앱 EC2와 Worker EC2만 허용
- 앱 EC2는 80/443만 공개하고 Worker EC2는 서비스 포트를 공개하지 않음

앱 서버:

```bash
docker compose --env-file .env -f compose/main.yml --project-directory . up --build -d --remove-orphans
docker compose --env-file .env -f compose/main.yml --project-directory . ps
curl -fsS http://localhost/health/
```

Worker 서버:

```bash
docker compose --env-file .env -f compose/worker.yml --project-directory . up --build -d --remove-orphans
docker compose --env-file .env -f compose/worker.yml --project-directory . ps
docker compose --env-file .env -f compose/worker.yml --project-directory . logs --tail=100 worker
```

합격 기준:

- 앱 서버에 PostgreSQL과 Worker 컨테이너가 없다.
- Worker 서버에는 Worker만 있다.
- 앱에서 생성한 작업을 별도 Worker가 RDS에서 선점·처리한다.
- 외부 접속은 Nginx를 통하고 FastAPI·ChromaDB에 직접 접속할 수 없다.
